Реалізовані ендпоінти:

POST /api/tasks

Створення нової задачі

Body: {title, description, priority, dueDate}

GET /api/tasks

Отримання всіх задач (сортування за createdAt)

GET /api/tasks/:id

Отримання задачі по ID

GET /api/tasks/status/:status

Фільтрація за статусом (pending/completed)

PUT /api/tasks/:id

Оновлення задачі

PUT /api/tasks/:id/complete

Позначити як виконану (markAsCompleted)

DELETE /api/tasks/:id

Видалення задачі

GET /api/tasks/summary

Агрегація: статистика по задачам NEW

Повертає: {total, pending, completed, completionRate}

Модель Task:

  • title (String, required)
  • description (String)
  • status (String: 'pending'/'completed')
  • priority (Number: 1-5)
  • dueDate (Date)
  • createdAt, updatedAt (Date, auto)
